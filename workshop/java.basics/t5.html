Java Coding Style

Compare the C-like version with a proper Java version, in the package basics.step5.java. See how much the code is cleaner, easier to read, and ultimately easier to understand? Comparer maintenant le code écrit à la mode du C avec le même code écrit correctement en Java. Le code se trouve dans le paquet basics.step5.java. Essayez de voir à quel point le code est plus propre, plus facile à lire, et au bout du compte plus facile à comprendre.

  • Look at the positive impact of having constructors. Regardez l'effet positif des constructeurs.
  • Look at how much better structured the entire code is, thanks to classes to group related methods together. Regardez à quel point le code est mieux structuré dans son ensemble, grace aux classes qui organisent les méthodes avec beaucoup de sens.
  • Look at how classes provide modularity and therefore clearer focus when reading or writing code. Regardez comment les classes sont une source de modularité du code et que donc l'attention du développeur est plus focalisé lors de la lecture ou l'écriture de code.

Each class goes in a file. Java advocates that each class goes in its own file, as a default. You will see soon enough that this rule has many exceptions. But for now, it is a good default. Chaque classe va dans un fichier. Java impose que chaque classe soit définie dans un fichier propre, par défaut. Vous verrez que cette règle a des exceptions. Mais pour l'instant, nous ne les considérons pas.

Separate the "main" class. To separate the "main" class is also a good default. The main class means the class that defines the entry point of the code, that is, the method main(String[]). For now, the main class is pretty much empty, but soon enough it will contain startup code, including the initialization of your application. It will have a rôle of its own and an independent class will be warranted. Séparer la classe "main". Séparer la classe "main" est aussi un bon pattern. La classe main est celle qui définit le point d'entrée du programme, c'est à dire, la méthode main(String[]). Pour l'instant, la classe main est quasiment vide, mais rapidement elle contiendra le code de lancement, incluant l'initialisation de votre application. Elle aura un rôle bien à elle, et une classe à part sera très bien pour cela.

Use constructors. Using constructors is really important because it helps to see how an object is initialized. The constructors give the different ways an object may be constructed. The code to construct an object is written once and easy to find, as opposed to be duplicated at every single new statement in your code. Avoiding redundant code is an important step towards cleaner, safer, and more robust code. Utiliser des constructeurs. Utiliser des constructeurs est vraiment important car cela aide à voir comment un objet est initialisé. Les constructeurs donnent les différents moyens d'initialiser un objet. Le code pour construire un objet est écrit une fois et ensuite utilisé autant que besoin, en opposition au fait de dupliquer le code d'initialisation à chaque fois qu'un objet est crée dans le code. Eviter le code redondant est un principe majeur pour obtenir du code propre, fiable et robuste.

Code Aesthetic

Learning the aesthetic of good programming is as important as learning the syntax or the grammar of a programming language. When you code using the C programming language, use the corresponding good practices. When you code using the Java programming language, use good object-oriented practices. Learning object-oriented good practices will take time, so don't put too much pressure on yourself right away, but you need to start paying attention from now on to the aesthetic of the classes you write as well as those written by others that you read. Apprendre à considérer l'esthétique d'une bonne programmation est aussi important que d'apprendre la syntax ou la grammaire d'un langage de programmation. Lorsque vous codez en C, utilisez les bonnes practiques qui ont été développé pour ce langage. Lorsque vous codez en Java, utiliser les bonnes pratiques de la programmation orientée-objet. Il vous faudra du temps pour les apprendre, ne vous mettez pas trop de pression tout de suite, mais il faut dès aujourd'hui pretter attention à l'esthétique des classes que vous écrivez ainsi qu'a celle des classes que vous lisez.
