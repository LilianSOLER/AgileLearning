Is Aliasing a Java Problem?

We are often asked by students if aliasing is a problem specific to Java or object-oriented programming. It is absolutely not a Java-specific challenge and not specific to object-oriented programming! As soon as you have references or pointers in a programming language, you have to consider aliasing. So developers in C or JavaScript or Python or C++ or C# all have to pay attention to aliasing. Unfortunately, aliasing is rarely taught and rarely understood properly. Nos étudiants nous demandent souvent si l'aliasing est un problème spécifique à Java ou la programmation orientée-objet. Ce n'est absolument pas le cas! Dès que vous avez des références ou des pointeurs dans un langage de programmation, vous devez maitriser l'aliasing. Malheureusement, l'aliasing n'est que trop rarement enseigné et trop souvent mal compris.

Also, remember that aliasing is not a problem. It is neither good or bad. Sometimes, aliasing is wanted, so much so that it would be really hard to do without. Think in terms of passing an object reference to a method. Of course you want aliasing between the caller and the callee. Think in terms of a list with double links (previous and next elements), aliasing is unavoidable there too. But sometimes, aliasing is the kiss of death because it is the source of some of the worst bugs. Rappelez vous, l'aliasing n'est ni bon ni mauvais. Parfois, il est important d'avoir des alias, parfois c'est un problème. Pensez au cas de passer un objet en argument à une méthode. Dans la majorité des cas, vous voulez avoir un alias et pas une copie. Si vous souhaitez une copie, vous pouvez toujours la faire avant l'appel. Pensez à des listes doublement chaînées, là encore, heureusement que l'aliasing est possible. Mais parfois, les problèmes d'aliasing sont la cause de bugs parmi les pires bugs qui soient.
