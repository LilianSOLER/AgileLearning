Eclipse Workbench

The Eclipse Workbench is the heart of the Human User Interface of the Eclipse platform, it plays an equivalent role for Eclipse to the role that Window Managers play for regular operating systems like Linux or Windows. The Eclipse Workbench is based on three essential concepts:

  • Perspectives
  • Views
  • Editors

A perspective in Eclipse is essentially a task-oriented screen organisation. For instance, Eclipse has a Java perspective that contains all the views and editors that Java developers need to write, edit, or format Java code. For debugging, Eclipse offers a different perspective, called the Debug Perspective, with views that are more specific to the task of debugging, such as a Debug view showing threads and call stacks or a Variables showing the values of local variables and arguments.

Eclipse Java Perspective

Look below at a typical organisation of the Eclipse Java Perspective.

Front and center, you see the editors. The Eclipse Java editor is one of the best Java editor available, with the most accurate syntax highlighting of the Java language. It comes with all the bells and whistles that one expects from a modern editor. It is of course configurable, in the Eclipse preferences.

Notice that you can open multiple editors, with each Java source in a different tab. But also, you can display multiple editors side by side. To do that, just grab a tab with your mouse (on the tab label) and drag it where you want it to be on the screen. In fact, you can do that with any view in Eclipse, a perspective is a grid layout and you can drag around any view to any place on the screen.

For instance, look on the left-hand side, we have several views in tabs. The currently selected tab is the Package Explorer, your Java browser in some sense. It shows the Eclipse Worbench from a Java perspective. You can see the list of your projects in the Workbench. If a project is a Java project, you can see the source folders and in each source folders you can seen the Java packages and then the Java classes. In other words, you see your code.

We have certain complementary views to the view "PackageExplorer" and the editor views.

  1. The view "Outline" that outline the currently selected class.
  2. The view "Problems" that lists all the problems in your workspace, that are usually compilation problems and project configuration problems.
  3. The view "Console" for the output of your running programs.
  4. The view "Search" for the results of your global textual searches, to start a search, look for the icon that looks like a handheld camping light.
