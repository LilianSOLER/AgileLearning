sprint=object.classes

This sprint is about classes and all the main advanced features of Java. So far, we have discussed object-oriented programming from an object perspective, focusing on objects, object references, and method invocations. But object-oriented programming is really two fold. The first fold is about objects, the second fold is about the hiearchy of classes and all the associated language features to help with advanced software design. This is where object-oriented programming really departs from traditional imperative programming like in C, proving you with powerful features that will help you handle complex programming tasks. Ce sprint est à propos des capacités avancées qu'offre Java pour écrire des classes. Jusqu'à maintenant, nous avons discuté de la programmation orientée-objet en se concentrant sur la dimension du graphe d'objets. Nous avons gardé l'écriture des classes au plus simple. Mais la programmation orientée-objet a bien deux dimensions: celle du graphe d'objets et celle de la hiérarchie des classes. En ce qui concerne les classes, Java propose des mécanismes avancées qui permettent d'aborder des conceptions logicielles plus avancées. C'est vraiment là que la programmation orientée-objet s'éloigne de la programmation impérative comme dans le langage C, en vous offrant des mécanismes puissants qui vont vous aider à mieux concevoir et mieux faire évoluer du code compliqué.

Like with everything that is new, it will feel a bit complex at first. You can choose to think it is only unnecessary complexity and choose to program with Java without using these features efficiently. But it would be like driving a race car in first gear because you do not want to learn how to stick shift. Learn the concepts, ask questions, and then put them into practice. Soon, you will wonder how you could have ever coded without these features. Comme en toutes choses, la nouveauté va vous paraître source de complexité. Vous pouvez en rester là et ne voir que cette complexité accrue et l'éviter lorsque vous programmez en Java. Mais cela revient à conduire une voiture de course toujours en première parce que vous ne voulez pas apprendre à passer les vitesses. Mais si vous apprenez les concepts, posez des questions, et que vous utilisez ces concepts dans vos programmes, alors vous verrez rapidement ce qu'ils apportent et vous vous demanderez comment vous avez fait pour programmer sans eux pendant si longtemps.

Reminder: do not forget to import the object.classes project in your Eclipse workspace. You also need to add to the build path the jar file that packaged your collections, the one you created at the end of the sprint "object.collections". Vous allez devoir importer le projet "object.classes" dans Eclipse. Il faudra aussi rajouter au "build path" l'archive de vos collections que vous avez créé à la fin du sprint "object.collections".
