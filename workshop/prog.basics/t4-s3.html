Making changes

The changes are adding methods so that our tests can drive your games programmatically, without requiring user inputs. Our tests will initialize your game with the already existing method: Les changements consistent à ajouter des méthodes de manière à ce que nos tests puissent diriger vos jeux programatiquement, sans attendre d'inout utilisateur. Nos tests sont initialiser vos jeux avec la méthode déjà existante: 

  public static void initGame(char first) { ... }

But then, our tests will use a few more methods that you need to implement. These methods are discussed below.

For the class HorizontalTicTacToe, the methods are the following: 

  public static boolean echoGame();
  public static boolean gameOver();
  public static char winner();
  public static boolean available(int row, int col);
  public static void play(int row, int col);

The method echoGame prints the board to the console (System.out).

The method gameOver returns true as soon as there is a winner or when the board is full.

The method winner returns a space (' ') if there is no winner or the actual winner (either 'X' or 'O').

The method available returns true if the cell at the given row and column is free and can therefore be played.

The method play request to play the cell at the given row and column. The cell is filled with the character of the player whose turn it is to play. This means that you keep track of player turns.

For the class VerticalTicTacToe, the methods are the following: 

  public static boolean echoGame();
  public static boolean gameOver();
  public static char winner();
  public static boolean available(int col);
  public static void play(int col);

The method echoGame prints the board to the console (System.out).

The method gameOver returns true as soon as there is a winner or when the board is full.

The method winner returns a space (' ') if there is no winner or the actual winner (either 'X' or 'O').

The method available returns true if the given column can be played, which means that it is not full.

The method play request to play the given column, dropping on that column the character of the player whose turn it is to play. This means that you keep track of player turns.
