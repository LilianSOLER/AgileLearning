So what is a string over an array of characters? A string is a sequence of characters, so an array seems a good fit. But should we force the array to have exactly the length of the string. Well, we could, but we will not. So we will assume that the string is a sequence of characters terminated with the "zero" character, the character '\u0000' in the Unicode encoding. Nous allons considérer qu'une chaine est une séquence de caractères terminée par le caractère "zero" (en encodage Unicode).

The character '\u0000' is not considered as being part of the string, it only indicates the end of the sequence of characters of the string. Comparing strings

Our last feature is the ability to compare strings. So again, we have three possible cases:

  1. The two strings are equal.
  2. The first string is smaller than the second number.
  3. The second number is smaller than the first number.

So again, we cannot use a boolean that has only two possible values: true or false. So we can use an integer to encode the three cases:

  1. Return 0 if the two numbers are equal.
  2. Return -1 if the first number is smaller than the second number.
  3. Return 1 if the second number is smaller than the first number.

But we need an order between strings in order to be able to speak of one string being less than another one. That order is called the lexicographic order. You already know it, it is a generalization of the alphabetical order. Nous avons besoin d'un ordre entre les chaînes pour être capable de dire qu'une chaîne est plus petite qu'une autre. Cet ordre est appelé l'ordre lexicographique.Vous le connaissez déjà, c'est une généralisation de l'ordre alphabétique.
