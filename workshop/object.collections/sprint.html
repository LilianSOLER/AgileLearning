sprint=object.collections

This sprint is about generic collections of objects, in the Eclipse project object.collections. One of the strength of object-oriented programming is the ability to provide generic classes that can be reused over and over in different programs. The classical collections are discussed in the different tasks below. Ce sprint parle de collections génériques d'objets. Une des forces de l'approche orientée-objet est la capacité d'offrir des classes génériques qui peuvent être utilisées et re-utilisées. Les tâches ci-dessous discutent des différentes collections que nous verrons dans ce sprint.

Java has a framework for all sorts of collections, so why bother developing our own collection classes? True, Java as a very complete framework, with a lot of features, but it is very complex. Our experience is that Java collections only make sense to advanced Java developers, others are misusing them more often than not. Once a developer has designed and coded his own collection classes, he has the right understanding to master the Java collections. Java a un canevas pour toutes sortes de collections, alors pourquoi s'embêter à développer les notres? C'est vrai, Java a une librairie de collections qui est très complète, mais qui est très complexe. Notre expérience est que les collections Java ne sont exploitable que par des développeurs Java avancés, sinon elles sont mal utilisées. Une fois que vous aurez développé vos propres collections, vous aurez la compréhension nécessaire pour utiliser les collections Java de façon appropriée.

Also, collections are a good way to discuss some important features of the Java language that we have not seen so far. In particular, we will discuss of the use of the class Object to make generic collections, collections that can hold any kind of objects, instances of any classes. Aussi, discuter de collections est une bonne façon de discuter de points importants du langage Java que nous n'avons pas encore abordé. En particulier, nous allons discuter l'utilisation de la classe Object pour faire des collections génériques, dans le sens de collections qui peuvent contenir des instances de n'importe quelles classes.

Note that we will reuse these collections, throughout the rest of the workshop. We will not use the official Java collections. Don't worry, you will learn them in time, when you are ready. Also, we believe that it is fulfilling for someone to reuse one's code. But it is also a bit challenging as you will need to fully debug your collections before you can trust using them. Notez que nous allons réutiliser ces collections, dans la suite de cet atelier. Nous n'allons pas utiliser les collections Java. Ne vous inquiétez pas, vous les apprendrez en temps et en heure, quand vous serez un peu plus avancé dans votre apprentissage de Java. Nous pensons aussi que c'est valorisant d'écrire du code et de le réutiliser.

That's all folks for this sprint.
